# 🧪 BlinkBot Comprehensive Testing Guide

This document outlines all the critical test cases required to ensure the BlinkBot (formerly RAGMate) platform is functioning correctly end-to-end. You can use the provided **Test Content** directly to validate the features.

---

## 1. 🤖 Agent Creation & Management (Studio)
**Objective**: Ensure users can create, configure, and manage their AI agents.

- [ ] **Create Basic Agent**: Navigate to the Studio, create a new agent with a custom system prompt, and verify it appears in the dashboard.
  - **Test Content (System Prompt)**: 
    > `You are an expert IT Support Agent for Acme Corp. Always answer politely, format your output in Markdown, and use the knowledge base provided to you.`
- [ ] **Provider Selection**: Test creating agents with different LLM providers (OpenAI, Groq, Ollama) and ensure their specific API keys are accepted and utilized.
- [ ] **Multi-Agent Setup**: Create a "Master" agent and assign sub-agents (e.g., Sales Agent, Support Agent) to it via the project configuration.
- [ ] **Agent Deletion**: Delete an agent and verify that its associated documents, chat history, and widget configurations are cascade-deleted from the database.

---

## 2. 📚 Knowledge Base & Data Ingestion (RAG)
**Objective**: Ensure documents and URLs are correctly chunked, vectorized, and stored.

- [ ] **PDF/TXT Upload**: Upload a PDF and a TXT file in the Knowledge Base tab. Verify the WebSocket progress bar completes successfully.
- [ ] **URL Web Scraping**: Submit a website URL via the `/process-url` feature and verify the text is extracted and chunked.
  - **Test Content (URL)**: `https://en.wikipedia.org/wiki/Retrieval-augmented_generation`
- [ ] **Google Drive Import**: Connect a Google Drive account, select a file, and verify it syncs to the agent's knowledge base.
- [ ] **Document Deletion**: Delete a document and verify its specific vector embeddings are removed from the `document_embeddings` table.

---

## 3. 💬 Chat Completion & Retrieval (MMR Pipeline)
**Objective**: Test the core conversational and retrieval engine.

- [ ] **Basic Chat**: Send a generic greeting and ensure the agent responds smoothly without attempting to search documents.
  - **Test Content (Prompt)**: `Hello! How are you today?`
- [ ] **Contextual Q&A**: Ask a specific question whose answer exists ONLY in an uploaded document. 
  - *Pass Condition*: The agent answers correctly and the Context Panel displays the exact chunks referenced.
- [ ] **MMR Diversity Check**: Ask a broad question covering a large document. Verify the retrieved chunks (visible in logs/Context Panel) are diverse and not just the same paragraph repeated 5 times.
- [ ] **Web Search Fallback**: Ask about a live current event. Verify the agent triggers the DuckDuckGo tool and returns live data.
  - **Test Content (Prompt)**: `What is the stock price of Apple today?`
- [ ] **Streaming**: Ensure the text streams in token-by-token rather than hanging and appearing all at once.

---

## 4. 🔀 Dynamic Multi-Agent Routing
**Objective**: Ensure the Master Coordinator accurately delegates tasks.

- [ ] **Intent Routing**: Chat with a Master Agent. Ask a question strictly related to a sub-agent's domain (e.g., "Sales"). 
  - *Pass Condition*: The UI displays `🤖 [Routed to: Sales Agent]` and the response is generated by the sub-agent.
  - **Test Content (Prompt)**: `I would like to purchase an enterprise license, what are the pricing tiers?`
- [ ] **Fallback Routing**: Ask a general question. Ensure the Master Agent handles it directly without routing to a specialized sub-agent.

---

## 5. 🔌 Custom Action Webhooks (API Manager)
**Objective**: Verify agents can communicate with external REST APIs.

- [ ] **Configure Connection**: Go to the API Connections Manager and add a new connection.
  - **Test Content (Base URL)**: `https://webhook.site/` *(Note: Visit webhook.site to get a unique temporary URL to paste here)*
  - **Test Content (Headers JSON)**: 
    ```json
    {
      "Authorization": "Bearer test-key-123",
      "Content-Type": "application/json"
    }
    ```
- [ ] **Tool Execution**: Prompt the agent to trigger the specific action defined in the tool.
  - **Test Content (Prompt)**: `Please trigger the test webhook to send a dummy payload.`
  - *Pass Condition*: Check the webhook.site page to verify the POST request was successfully received with the correct headers.

---

## 6. 🌐 Embeddable Widget
**Objective**: Ensure external deployments function correctly.

- [ ] **Widget Generation**: Go to the Deploy tab and copy the HTML `<script>` snippet.
- [ ] **Local HTML Test**: Paste the snippet into a blank HTML file and open it in a browser. Ensure the chat bubble appears.
  - **Test Content (index.html)**:
    ```html
    <!DOCTYPE html>
    <html lang="en">
    <head>
        <meta charset="UTF-8">
        <title>Widget Test</title>
    </head>
    <body style="background-color: #f4f4f5; height: 100vh;">
        <h1 style="padding: 20px;">My Website Testing Ground</h1>
        <!-- PASTE YOUR COPIED WIDGET SCRIPT BELOW -->
        
    </body>
    </html>
    ```
- [ ] **Cross-Origin Chat**: Send a message via the embedded widget and verify it successfully hits the backend (`/api/widget`) without CORS errors.
- [ ] **Quota Limits**: Exceed the monthly message limit for the widget and verify it gracefully blocks further messages with an upgrade prompt.

---

## 7. 💻 Developer API
**Objective**: Test headless programmatic access.

- [ ] **API Key Generation**: Generate a new Developer API Key in the Settings page.
- [ ] **cURL Request**: Run a terminal `curl` command to `/api/v1/chat` using the generated `x-api-key`.
  - **Test Content (Terminal Command)**:
    ```bash
    curl -X POST http://localhost:8000/api/v1/chat \
         -H "Content-Type: application/json" \
         -H "x-api-key: rm_YOUR_COPIED_KEY_HERE" \
         -d '{"message": "Hello! What can you help me with?"}'
    ```
  - *Pass Condition*: The API returns a successful streaming response or a valid JSON payload.
- [ ] **Authentication Block**: Attempt the same `curl` request with an invalid API key and verify it returns a `401 Unauthorized`.

---

## 8. 🏢 Team Workspaces & RBAC (Role-Based Access Control)
**Objective**: Ensure data isolation and security.

- [ ] **Workspace Isolation**: Create Workspace A and Workspace B. Verify that agents/chats from A are completely invisible when logged into B.
- [ ] **Member Invitation**: Invite a new user as a "Viewer". 
- [ ] **Permission Enforcement**: Log in as the "Viewer" and attempt to delete an agent or view billing settings. Ensure the UI blocks the action or the API returns a `403 Forbidden`.
- [ ] **Real-Time Notifications**: Have one user submit "Feedback" on a chat message. Ensure the Workspace Owner immediately receives a notification bell ping without refreshing the page.

---

## 9. 🔌 Custom Action Webhooks (E-Commerce Use Case)
**Objective**: Ensure the agent can trigger external APIs to take actions (e.g., track an order, place an order).

- [ ] **Setup Webhook Receiver**:
  - Go to `https://webhook.site` and copy your unique URL.
- [ ] **Configure the Agent Endpoint**:
  - Open the Agent Settings in the Studio.
  - Go to the **API Endpoints** tab.
  - **Base URL**: `[Your Webhook.site URL]`
  - **Endpoint Name**: `Track Order`
  - **Method**: `POST`
  - **Path**: `/track`
  - **Description**: `Use this tool to track an order when the user provides an order ID.`
  - **Payload Format (JSON)**: `{"order_id": "{id}"}`
  - Save the agent settings.
- [ ] **Test the Action**:
  - Start a chat with the agent in the UI.
  - **Test Content (User Message)**: "Can you track my order #ORD-12345?"
  - *Pass Condition*: 
    1. The agent responds confirming it is tracking the order or has triggered the system.
    2. A `POST` request instantly appears in your `webhook.site` dashboard with the JSON payload: `{"order_id": "ORD-12345"}`.
